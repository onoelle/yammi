yammi - Howto
-------------

CVS-Howto
---------
dies kreiert lediglich den account auf cvs.sourceforge
ssh -l onoelle cvs.yammi.sourceforge.net

import: bob -> cvs@sourceforge
cvs -d:ext:onoelle@cvs.sourceforge.net:/cvsroot/yammi import kyammi vendor start

checkout: cvs@sourceforge -> bob
cvs -d:ext:onoelle@cvs.sourceforge.net:/cvsroot/yammi co kyammi



VI essentials
-------------
ESC + :w    save file
ESC + :q    quit vi


SmokeTest
---------
1. delete all installed files (rm -r /opt/kde/share/apps/yammi, rm -r /home/oliver/.kde/share/config/yammirc)
2. fresh install of yammi
3. check that files get installed in correct places
4. startup without any existing configuration files or database
5. scan harddisk, save database
   - create an empty category, create a category and insert 2 songs
6. exit yammi

7. start again
8. verify that database+categories were correctly saved
9. with all media players:
   - empty playlist
   - enqueue song
   - start playing
   - enqueue song
   - enqueue as next
   - play now
   - stop, start
   - pause
   - ...
   


prepare Release
---------------
check:
- check yammi version in configure.in(.in)
- AUTHORS/CREDITS
- README
- TODO/CHANGELOG
- start program and check output: any couts/kdDebugs() that should be deleted?
- remove all ~ and .orig / .rej files (from applying patches)
- change QT version 3.3 => 3.2 in .ui files
- try to check and fix i18n strings as far as possible
- execute smoke tests



In kdevelop
1. Build -> distclean
2. Build -> Run automake & friends
3. Build -> Run configure
4. Tools -> Project Distribution & Publishing -> make distribution

anschliessend CVS taggen!
